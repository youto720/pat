# pat — ゲーム開発プロジェクト

## プロジェクト概要

**pat** は、グリッド状のパネルを指（またはマウス）でなぞっていく一筆書きパズルゲームです。
スタート位置からゴール位置まで、できるだけ多くのマスを通りながらたどり着くことを目指します。

## 技術スタック

- **フロントエンド**: React + TypeScript + Vite
- **スタイリング**: Tailwind CSS + CSS Modules（アニメーション部分）
- **音響**: Web Audio API（外部ライブラリ不要）
- **アニメーション**: CSS アニメーション + Canvas API（エフェクト）
- **デプロイ**: Vercel または Netlify（静的ホスティング）
- **将来的なスマホ対応**: Capacitor.js でラップしてネイティブアプリ化

## ゲーム仕様（詳細）

### グリッド

- 最初は **4×6** 程度の小さいグリッドから開始
- ゴールするたびにグリッドサイズが段階的に拡大（最大 8×12 程度）
- マスは正方形。画面幅に応じてレスポンシブにサイズ調整

### スタート（S）とゴール（G）

- スタートは **オレンジ色**（`#F4845F` 系）
- ゴールは **ピンク/サーモン色**（`#E87070` 系）
- スタートマスを押した瞬間からなぞり開始
- ゴールマスに到達した瞬間にゲームクリア

### 操作ルール

1. スタートマスから指（マウス）を離さずなぞり続ける
2. 一度通ったマスは再度通れない（重複不可）
3. 上下左右の隣接マスにのみ移動可能（斜め不可）
4. 以下の場合はリセット（やり直し）：
   - 指を途中で離した場合
   - 一度通ったマスを再度なぞろうとした場合
   - ゴール以外の場所で操作を終了した場合

### スコア

- 1マスなぞるごとに **+10点**
- タイム bonus: 残り時間 × 係数（将来実装）
- 累積スコアは localStorage に保存

### レベルアップ条件

- ゴール達成ごとにカウント
- 3回ゴール → グリッドサイズ +1列
- 5回ゴール → グリッドサイズ +1行
- （調整可能）

### 視覚的フィードバック

| 状態 | 色 |
|------|-----|
| 未訪問マス | 白 / 薄いグレー |
| なぞり中マス | グリーン系（`#4CAF7D` 系）|
| スタートマス | オレンジ（`#F4845F`）|
| ゴールマス | ピンク（`#E87070`）|
| リセット時 | 全マス短いフェードアウトで白に戻る |

### 音（Web Audio API で生成）

- **マスをなぞる音**: 短い「ポン」音。音程はなぞるごとに少し上がる（気持ちよさ重視）
- **ゴール音**: 明るいファンファーレ風の音（3音程度）
- **リセット音**: 低めのシュッという音
- 音はすべて Web Audio API で合成（mp3 ファイル不要）

### ゴールエフェクト

- パーティクルが四方に広がる（Canvas）
- スコア加算のアニメーション表示（+○○ が浮き上がる）
- グリッド全体が一瞬光る（CSS）
- 「GOAL!」テキストが中央にポップアップ

## UIデザイン方針

スケッチ（`design/sketch.jpg`）を参照。

- **ヘッダー**: タイトル「slide」（太字）、左にハンバーガーメニュー、右にスコア表示
- **タイマー**: グリッド上部に `⏱ 00:00` 形式（将来実装）
- **グリッド**: 画面中央に配置、角丸の外枠
- **全体トーン**: クリーン・ミニマル。背景は白または極薄グレー
- **フォント**: システムフォントではなく、`Nunito` または `Fredoka One`（丸みのある親しみやすい書体）

## デプロイ・公開

```bash
# ビルド
npm run build

# Vercel でのデプロイ（推奨）
vercel deploy --prod
```

- カスタムドメインは Vercel のダッシュボードから設定
- SSL は Vercel が自動で付与（Let's Encrypt）
- 将来的に Capacitor.js でスマホアプリ化：
  ```bash
  npx cap add ios
  npx cap add android
  npx cap sync
  ```

## 開発の進め方（優先順位）

1. **Phase 1**: グリッド表示 + なぞり操作 + 基本ルール
2. **Phase 2**: スコア・レベルアップ・音
3. **Phase 3**: ゴールエフェクト・アニメーション
4. **Phase 4**: タイマー・ハイスコア保存
5. **Phase 5**: Capacitor.js でスマホアプリ化

## 注意事項

- **タッチ操作を最優先**で実装（`touchstart`, `touchmove`, `touchend`）
- マウス操作も同等に動作させる（PC でのテスト用）
- `passive: false` で `touchmove` のデフォルト（スクロール）を防ぐ
- iOS Safari での `-webkit-tap-highlight-color` 対策を忘れずに
- グリッドは `user-select: none` にしてテキスト選択を防ぐ
